## 5. 服务配置

### 5.1 systemd

服务文件通常位于:

```bash
/etc/systemd/system/football-analyst.service
```

建议配置保留 `DB_PATH`，避免模块各自猜路径:

```ini
[Unit]
Description=Football Analyst API Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/football_tools
Environment=PATH=/opt/football_tools/venv/bin:/usr/bin:/bin
Environment=PYTHONPATH=/opt/football_tools:/opt/football_tools/backend
Environment=DB_PATH=/opt/football_tools/data/football_v2.db
Environment=TZ=Asia/Shanghai
ExecStart=/opt/football_tools/venv/bin/uvicorn backend.app.main:app --host 0.0.0.0 --port 8000 --workers 2
Restart=always
RestartSec=5
StandardOutput=append:/opt/football_tools/logs/uvicorn.log
StandardError=append:/opt/football_tools/logs/uvicorn_error.log

[Install]
WantedBy=multi-user.target
```

常用命令:

```bash
systemctl status football-analyst
systemctl restart football-analyst
systemctl stop football-analyst
journalctl -u football-analyst -f --no-pager -n 50
```

### 5.2 Nginx

服务前缀已经通过公网验证为:

```text
http://1.117.70.20/api/v1/...
```

因此 Nginx 应该保留 `/api/...` 路径转发给 FastAPI，不建议使用会 strip `/api/` 的配置。

推荐形式:

```nginx
server {
    listen 80;
    server_name 1.117.70.20;

    location / {
        root /opt/football_tools/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;
    }
}
```

验证:

```bash
nginx -t
systemctl reload nginx
curl http://127.0.0.1:8000/api/v1/health
curl http://1.117.70.20/api/v1/health
```

```bash

```

```text
/opt/football_tools/frontend/dist/
```

Nginx 直接服务该目录。仅前端更新时，一般无需重启后端。

### 8.3 手动触发采集/分析

```bash
# 今日同步
curl -X POST "http://127.0.0.1:8000/api/v1/lottery/sync"

# 单场强制分析
curl -X POST "http://127.0.0.1:8000/api/v1/lottery/analyze/MATCH_ID?force=true&sync=true"

# 批量分析指定日期
curl -X POST "http://127.0.0.1:8000/api/v1/lottery/analyze-batch?force=true" \
  -H "Content-Type: application/json" \
  -d '{"date":"2026-06-18"}'

# 完整管道
curl -X POST "http://127.0.0.1:8000/api/v1/lottery/pipeline?backfill_days=7&force_analyze=true"
```

### 8.4 数据库只读检查

```bash
sqlite3 /opt/football_tools/data/football_v2.db
```

常用 SQL:

```sql
SELECT 'matches', COUNT(*) FROM matches UNION ALL
SELECT 'teams', COUNT(*) FROM teams UNION ALL
SELECT 'lottery_matches', COUNT(*) FROM lottery_matches UNION ALL
SELECT 'lottery_odds', COUNT(*) FROM lottery_odds UNION ALL
SELECT 'lottery_results', COUNT(*) FROM lottery_results UNION ALL
SELECT 'lottery_validation', COUNT(*) FROM lottery_validation;

SELECT source_name, source_category, status, last_success, last_failure, failure_count, success_rate
FROM data_source_health
ORDER BY source_name;

SELECT play_type, snapshot_type, COUNT(*) AS c, MAX(update_time) AS latest
FROM lottery_odds
GROUP BY play_type, snapshot_type
ORDER BY c DESC;

SELECT lm.lottery_match_id, lm.match_date, lm.home_team_cn, lm.away_team_cn, lm.oddsfe_event_id
FROM lottery_matches lm
LEFT JOIN lottery_results lr ON lm.lottery_match_id = lr.lottery_match_id
WHERE lm.sell_status IN ('closed', 'finished')
  AND lr.lottery_match_id IS NULL
ORDER BY lm.match_date DESC;
```

### 8.5 数据库备份

结构调整前必须备份:

```bash
mkdir -p /opt/football_tools/backups
sqlite3 /opt/football_tools/data/football_v2.db ".backup '/opt/football_tools/backups/football_v2_$(date +%Y%m%d_%H%M%S).db'"
```

大文件同步建议不要覆盖式直接推送，优先:

- 本地生成迁移脚本。
- 云端备份数据库。
- 云端执行迁移。
- 验证接口和关键 SQL。

## 10. 已知问题与待改进

### 10.1 磁盘空间

磁盘使用率约 79%，主库约 1.4 GB。需要:

- 定期清理日志。
- 建立数据库备份保留策略。
- 后续考虑历史数据冷归档，而不是直接删除。

### 10.2 内存

1.9 GB 内存运行 2 个 uvicorn worker 偏紧。高峰期需观察:

```bash
free -h
ps aux --sort=-%mem | head -10
```

### 10.3 odds 快照

当前 odds 是强项，但还要补:

- opening / current / latest / closing 的标准定义。
- CLV 采集任务稳定化。
- 盘口变化轨迹表。
- 相同盘口、相同赔率结构的历史案例检索。

### 10.4 玩法复盘

目前 SPF 样本最多，OU 有少量样本，RQSPF/BF/BQC 明显不足。下一步:

- 对所有已结束比赛补全 RQSPF/BF/BQC 赛果。
- 分玩法写入 `lottery_validation`。
- 按玩法统计准确率、回报、场景归因。

### 10.5 世界杯模块上云

本地已有世界杯 2026:

- 小组积分
- 第三名排名
- 晋级规则
- 32 强淘汰路径
- 单场世界杯上下文

云端仍需部署并验证:


### 10.6 数据内容层

长期目标不是只靠 odds，而是:

```text
odds 主轴 + 阵容/伤停/新闻/天气/赛制/战意证据 + 赛后复盘 + 相似比赛库
```

优先补齐:

- `collection_runs`
- `source_artifacts`
- `source_entity_mappings`
- `match_feature_snapshots`
- `match_context_snapshots`
- `post_match_reviews`
- `similar_match_cases`
