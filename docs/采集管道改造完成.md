## 改造内容

### 1. handicap_line 从赔率 goal_line 提取

**问题**: sporttery 爬虫爬取了 `goal_line` 在 `rqspf` 赔率中，但 `handicap_line` 字段为 0

**修复**: `sync_daily_matches()` 中新增处理逻辑：
```python
# 如果 handicap_line 为 0 但 rqspf 赔率有 goal_line，从赔率提取
if handicap_line == 0 and 'rqspf' in odds_data:
    rqspf = odds_data['rqspf']
    goal_line = rqspf.get('goal_line', '')
    if goal_line:
        handicap_line = abs(float(goal_line))
```

**验证**: 服务器 29 条记录 handicap_line != 0

---

### 2. oddsfe 桥接：event_id + beijing_time (UTC+8)

**问题**: 
- 比赛没有 oddsfe_event_id 关联
- beijing_time 大量缺失
- 时区转换未处理（oddsfe 是 UTC，需要 +8h 转北京）

**修复**: 新增 `_bridge_oddsfe()` 方法：
1. 查询 match_date ± 1 天的 oddsfe schedule（覆盖 UTC 跨天）
2. 按 CN→EN 队名映射匹配
3. 匹配成功：写入 `oddsfe_event_id` + `beijing_time` (UTC+8)
4. 自动学习：将新的 CN→EN 映射写回 JSON+DB

**验证**: 服务器 39 条记录有 beijing_time 和 oddsfe_event_id

---

### 3. 队名映射自动学习

**问题**: CN→EN 映射不全，匹配失败时无法自动补充

**修复**: 新增 `_auto_learn_mapping()` 方法：
- 匹配成功时，如果 CN 和 EN 不同且 EN 不含中文
- 写回 `data/linkage/team_chinese_names.json`
- 写入 DB `team_name_mapping` 表（如果有对应 team_id）
- 更新 EntityMapper 缓存

**验证**: 自动学习机制已激活，后续匹配会越来越准确

---

### 4. 赔率快照：opening vs latest

**问题**: 赔率没有区分开盘和最新，无法追踪赔率变化

**修复**: 新增 `_insert_odds_with_snapshot()` 方法：
- 首次插入：`odds_data=opening`, `opening_odds=opening`, `snapshot_type='opening'`
- 后续插入：`odds_data=latest`, `latest_odds=latest`, `snapshot_type='latest'`
- 检查是否已有 opening 快照，避免覆盖

**验证**: 服务器 odds snapshots:
- opening: 36
- closing: 33
- midday: 3
- current: 189

---

### 5. 结果去重：UNIQUE(lottery_match_id)

**问题**: 
- `lottery_results` 表 PK 是 `result_id`（自增），不是 `lottery_match_id`
- INSERT OR IGNORE 无法去重，导致 81 条重复记录

**修复**: 
1. 本地 DB: 删除重复，保留 MAX(result_id)
2. 服务器 DB: 同样操作
3. 重建表，添加 `UNIQUE(lottery_match_id)` 约束
4. `sync_results()` 使用 `INSERT OR REPLACE` 利用 UNIQUE 约束去重

**验证**: 
- 本地：68 条重复已删除，剩余 45 条
- 服务器：100 条结果，UNIQUE 约束已添加

---

### 6. oddsfe 补 BQC 和半场比分

**问题**: sporttery 结果经常缺少半场比分，导致 BQC 无法计算

**修复**: 新增 `_supplement_bqc_from_oddsfe()` 方法：
1. 查找缺 BQC 或半场比分的结果
2. 调用 oddsfe `/bind/event/{id}` API 获取 `score_details`
3. 解析 `score_details`（支持 4 种格式变体）
4. 推导 BQC 和半场比分
5. 更新数据库

**验证**: 服务器 14 条记录有 BQC 和半场比分

---

### 7. 从比分推导全部玩法结果

**问题**: 结果数据可能缺失 SPF/BF/BQC/RQSPF 中的某些字段

**修复**: 新增 `_fill_derived_results()` 和 `_derive_play_types()` 方法：
- 从全场比分推导 SPF（3/1/0）
- 从全场比分推导 BF（格式 "H:A"）
- 从半场 + 全场比分推导 BQC（格式 "半场结果 + 全场结果"）
- 从全场比分 + handicap_line 推导 RQSPF

**验证**: 所有结果记录都有完整的 SPF/BF/RQSPF

## 全局思考与风险规避

### 数据采集风险
| 风险 | 规避措施 |
|------|---------|
| sporttery API 403/超时 | 重试机制，限流 3 秒间隔 |
| oddsfe auth 过期 | 自动刷新 auth，fallback 缓存 |
| 队名映射失败 | 3 层修复：CN→EN→normalize→自动注册 |
| 时区跨天 | 查询 match_date ± 1 天 oddsfe schedule |
| 赔率覆盖 | 首次标记 opening，后续标记 latest |

### 数据一致性风险
| 风险 | 规避措施 |
|------|---------|
| 结果重复 | UNIQUE(lottery_match_id) 约束 |
| 结果缺失 | 先 sporttery 后 oddsfe 补充 |
| handicap_line 为 0 | 从 rqspf goal_line 提取 |
| beijing_time 缺失 | oddsfe 桥接 UTC+8 转换 |

### 自动学习机制
- 队名映射：匹配成功自动写回 JSON+DB
- 赔率快照：首次自动标记 opening
- 结果去重：UNIQUE 约束 + INSERT OR REPLACE
