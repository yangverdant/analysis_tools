## 用户画像

- **角色**: 技术型产品人——既理解系统架构, 又关注产品效果
- **核心诉求**: 自动获取未来赛事→自动分析→自动复盘→自动优化迭代→循环往复(原话)
- **沟通风格**: 直接, 不容忍推一下动一下; 期望系统自驱动, 不需要人推
- **技术理解**: 能读懂代码, 能做架构决策(如"别接sporttery了"决定切换oddsfe)
- **验收标准**: 系统自己跑, 不需要人工干预; 用户只看结果(前端/推送)

## 如何协作

- 不要修单个bug就停——修完要验证整个闭环通不通
- 汇报时给出全局状态(全链路覆盖率), 不是单点修复说明
- 设计决策时考虑"系统无人值守能跑多久"
- 推送和前端是用户接触点, 优先保证这两处的数据正确和中文显示
