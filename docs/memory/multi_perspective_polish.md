## 8视角打磨汇总

### 主教练视角
- form权重不看赛程密度(3天2场vs7天1场含金量不同) → 加schedule_density因子
- 主客场分析不分对手强度(主场对强队vs弱队完全不同) → form加对手Elo加权
- 动机分析太粗(保级vs争冠殊死一搏 vs 保级vs中游可能泄气) → 加motivation_matchup
- 杯赛轮换预测缺失 → 加rotation_risk评估

### 球员视角
- 伤病信息几乎为零(apifootball友谊赛无数据) → 8:00加首发阵容预期推断
- 疲劳只看比赛数不看出场时间 → 中期改进，短期标注low
- 红黄牌停赛不考虑 → 8:00 intel加suspensions查询
- 国家队征召影响俱乐部 → 加FIFA比赛日历检测

### 管理者视角
- 没有阵容价值维度 → 中期采集Transfermarkt身价
- 没有赛季阶段意识 → MatchProfile加season_phase: early/mid/late/playoff
- 没有投资回报视角 → 复盘加球队价值低估/高估统计

### 资深从业者视角
- 裁判因素完全忽略 → 中期采集裁判数据
- 德比/宿敌没有权重 → 德比战特殊处理(降form/Elo权重，加rivalry权重)
- 赛季初假强队问题(form被对手弱队欺骗) → form加对手强度加权
- 默契球无法检测 → 长期：动机+赛程交叉分析

### 企划家视角(关键!)
- 没有投资回报追踪 → 加BetRecord+ROI计算
- 没有资金管理 → 加BankrollManager(初始10000元，Kelly投注)
- 没有按玩法统计ROI → 复盘按play_type分别计算
- 没有"不买"选项 → 推送加"今日无价值投注"

### 使用者视角(关键!)
- 推荐太多不知道选哪个 → 只推TOP3价值投注(Kelly排序)
- BF/BQC推荐没具体赔率 → 推送必须含体彩赔率+模型概率+edge
- 没有止损概念 → 加周度止损线(亏损>30%→降Kelly或暂停)
- 看不懂分析术语 → 推送用自然语言一句话理由

### 观众视角
- 前端没有分析师视角 → 加分析详情页+因子分解饼图
- 没有历史战绩可视化 → 加准确率趋势图
- 没有翻车故事 → 复盘页展示归因

### 用户视角
- 手动干预太多 → 日循环全自动，只在报警时手动
- 报错看不懂 → 错误翻译为用户语言
- 没有一键回测 → 加回测按钮
- 冷启动太复杂 → 首次运行自动热启动+进度条
- 部署太复杂 → docker-compose.yml + .env模板 + 一键启动

## 优先级调整

企划家和使用者视角的改动提升到P1：
- 投资回报追踪(BetRecord+ROI)
- TOP3筛选+自然语言理由
- 周度止损线

总工期从20天调整到25天
