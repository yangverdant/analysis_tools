## 规则: 修复必须看全局闭环, 不要修单个bug就停

**Why**: 用户原话"说实话你真的我想骂你 非得搓一下动一下?" + "我之前想要什么效果 自动获取未来赛事比赛内容 自动分析 自动复盘 自动 优化迭代 循环往复啊". 我之前修了CN enum泄漏就停了, 但用户要的是完整自动化闭环运转. 修一个bug时没检查这个bug是否阻断整个闭环, 导致修完CN问题但分析/学习/推送三个环节都还是断的.

**How to apply**:
1. 每次修bug前, 先问"这个bug是否阻断闭环?" 如果是, 继续追踪闭环的其他断点
2. 修完一个bug后, 立即验证整个闭环(采集→分析→推送→复盘→学习)是否通, 不要只验证修的那一点
3. 用户说"自动获取→自动分析→自动复盘→自动优化迭代→循环往复", 这是验收标准: 不需要人推一把, 系统自己跑
4. 当发现多个P0断点时, 一次性全部修完再汇报, 不要修一个汇报一个等用户推
