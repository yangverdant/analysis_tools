## 2. 总目标

把系统从“能自动跑”推进到“能解释地自动变强”：

```text
采集 -> 分析 -> 复盘 -> 发现缺口 -> 自动补数据 -> 重新分析未赛 -> 记录变化 -> 回测验证
```

最终要做到：

1. 每场比赛知道缺什么数据。
2. 每个缺口知道候选数据源。
3. 每个数据源知道健康状态和失败原因。
4. 采集成功后留下 source_artifacts。
5. 采集失败后有降级方案。
6. 复盘错误能反推出下一次要补的数据。
7. 学习后能影响未开赛比赛。
8. 前端能看到整个链路，而不是只看到“缺情报”。

重要边界：

```text
世界杯只是当前重点，不是系统边界。
后续必须覆盖每天滚动更新的所有可分析比赛。
```

系统每天都要处理四类比赛：

```text
新同步进来的未开赛比赛
今天/明天即将开赛的比赛
正在进行或刚结束、等待赛果补齐的比赛
已经结束、需要复盘和反哺模型的比赛
```

不能只围绕世界杯做一次性补齐。新的比赛每天都会进入库里，自动化必须持续识别新比赛、补采基础信息、生成分析、前端展示状态，并在赛后自动复盘。

## 2.1 全赛事滚动链路目标

### 每天必须自动发生的事情

```text
1. 同步今日/明日/未来窗口赛事
2. 识别新增未开赛比赛
3. 建立或更新基础比赛信息
4. 补赔率、盘口、真实 O/U、赛程时间、球队映射
5. 补情报包：伤停、预计首发、新闻、天气、赛程疲劳
6. 补基础分析因子：近期状态、攻防节奏、历史交锋、ELO/FIFA、主客/中立、赛事动机
7. 生成赛前分析
8. 前端展示：是否可分析、缺什么、正在采什么、分析是否最新
9. 比赛结束后补赛果、半场、玩法结果
10. 自动复盘命中/错误
11. 错误归因生成下一轮数据需求
12. 学习后重分析未来未开赛比赛
```

### 不同赛事的处理差异

| 赛事类型 | 重点因素 | 注意点 |
|---|---|---|
| 世界杯/国家队大赛 | 小组形势、晋级压力、国家队实力、旅行休息、阵容完整度 | 小组赛只有 3 场，淘汰赛规则不同 |
| 国家队友谊赛 | 轮换、试阵、动机不稳定、名单质量 | 历史结果参考要降权 |
| 世界杯预选赛/洲际赛事 | 积分压力、赛程、主客旅行、阵容征召 | 动机和积分形势权重要提高 |
| 联赛 | 长期攻防状态、主客场、伤停、赛程密度 | 可用历史样本更多 |
| 杯赛 | 轮换、淘汰赛动机、赛制、加时点球 | 常规联赛模型不能直接套 |

验收标准：

```text
不管今天同步进来的是世界杯、友谊赛、联赛还是杯赛：
系统都能识别赛事类型，补齐对应基础因素，并在前端显示采集/分析状态。
```

## 9. P0-6：分析模型重构，不再只做表面复盘

### 当前问题

云端 `lottery_validation` 的真实复盘结果说明：当前分析效果不合格。

截至本次复核，按玩法统计：

```text
spf    胜平负      179 场  正确 98   准确率 54.7%
ou     大小球      155 场  正确 81   准确率 52.3%
rqspf  让球胜平负  146 场  正确 72   准确率 49.3%
bqc    半全场      100 场  正确 39   准确率 39.0%
bf     比分        152 场  正确 37   准确率 24.3%
```

按场景统计也不好：

```text
international_cup  365 条  46.8%
league             330 条  41.8%
friendly_intl       37 条  48.6%
```

置信度也没有完全校准：

```text
low<55     507 条  39.3%
mid55-75   134 条  59.0%
high>=75    25 条  68.0%
```

错误归因质量更差：

```text
unknown 680 条，其中错误 353 条
```

结论：

```text
当前模型不是“差一点”，而是分析因子、玩法联动、置信度校准、错误归因都不够。
```

### 根本问题

现在的分析有几个明显问题：

1. 胜平负、让球、大小球、比分、半全场之间经常没有形成一致的比赛脚本。
2. 大小球和比分没有稳定共享同一个进球预期和盘口逻辑。
3. 半全场没有充分使用上半场进球/丢球/节奏画像。
4. 让球胜平负没有稳定围绕“赢几球/输几球边界”推导。
5. 比分候选常常像附属展示，而不是由方向、进球、双方进球率、盘口共同推导。
6. 不同赛事类型没有足够分流，国家队、友谊赛、杯赛、联赛不能共用一套权重。
7. 低置信比赛仍然给出像高置信一样的推荐，前端观感会像“瞎猜”。
8. 错误归因 unknown 太多，说明复盘还没有真正解释错因。

### 要做

#### 9.1 建立基准线

所有模型优化必须先和基准线比较。

基准线至少包括：

```text
market_favorite_baseline      赔率最低方向
market_implied_baseline       赔率隐含概率
poisson_baseline              纯泊松
elo_baseline                  纯 ELO/FIFA
recent_form_baseline          近期状态
hybrid_current_model          当前模型
```

验收：

```text
如果新模型跑不过 market_favorite_baseline，不允许标为优化。
```

#### 9.2 重建比赛脚本层

每场比赛先生成一个统一的 match script：

```json
{
  "direction_axis": "home_edge",
  "margin_axis": "home_by_1_or_2",
  "goal_axis": "2_to_3_goals",
  "btts_axis": "away_can_score",
  "first_half_axis": "slow_first_half",
  "late_goal_risk": "medium",
  "uncertainty": "high",
  "key_drivers": ["market", "attack_defense", "lineup_gap", "motivation"]
}
```

然后五个玩法都从这个脚本推导：

```text
胜平负 -> 方向轴
让球胜平负 -> 边界轴
大小球 -> 总进球轴
比分 -> 方向 + 总进球 + BTTS + 边界
半全场 -> 半场节奏 + 全场方向
```

验收：

```text
不能出现：
胜平负看主胜
让球看主队-1让胜
比分首选却是1-1
半全场却是负负
```

#### 9.3 补齐分析因子

每场比赛至少要有这些分析面：

```text
基础实力：ELO/FIFA/市场赔率
近期状态：近5-10场胜平负、进球、失球
进攻能力：场均进球、射门/xG可用时使用
防守强度：场均失球、零封、被射门/xGA可用时使用

半场画像：上半场进球、上半场失球、早球率
让球边界：赢1球/赢2+球/输盘率
阵容伤停：关键球员缺失、门将/中卫/前锋影响
赛程疲劳：休息天数、旅行、连续作战
赛事动机：积分、晋级、淘汰、轮换、友谊赛
赛事类型：联赛/杯赛/国家队/友谊赛/淘汰赛
盘口变化：开盘、即时盘、异常变动
相似案例：同盘口、同方向、同赛事类型、同强弱差
```

验收：

```text
分析详情里每个关键判断都能追到至少一个证据源或 fallback 标记。
```

#### 9.4 置信度重校准

当前高置信样本少，且 high>=75 只有 68%。

