```
┌─────────────────────────────────────────────────────────────┐
│                      云服务器（主）                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐     │

│  │ (每日运行)  │    │ (脚本处理)  │    │ (标准化数据) │     │
│  └─────────────┘    └─────────────┘    └─────────────┘     │
│         ▲                                      │            │
│         │                                      ▼            │
│  ┌─────────────┐                       ┌─────────────┐     │
│  │ 数据源      │                       │ API接口     │     │

         ▲
```

## 三种数据来源的处理方式

### 1. 自动爬虫数据（主要来源）

**适合场景**：联赛数据、常规比赛结果、赔率更新

```bash
# 云服务器定时任务
# 每天自动抓取更新

# /home/football_tools/scripts/cron_daily_update.py
# - 抓取前一天比赛结果
# - 更新赔率数据
# - 检测数据异常
```

### 2. 半手动上传数据

**适合场景**：新赛事数据、历史数据补全、特殊数据

```bash
# 本地整理 → 上传到云端

# 步骤：
# 1. 本地按模板整理CSV
# 2. 运行本地清洗脚本验证
# 3. 上传到云端指定目录
# 4. 云端脚本自动合并到new_data/
```

### 3. 手动录入数据

**适合场景**：2026世界杯等未开赛数据、预测数据

```bash
# 直接在云端编辑或通过Web界面录入
```

## 具体实现

### 方案A：简单版（推荐起步）

```
云服务器职责：
- 存储所有数据
- 运行定时爬虫
- 提供数据访问

本地电脑职责：
- 复杂数据整理
- 新数据模板制作
- 脚本开发和测试
```

**同步工具**：rsync / WinSCP

### 方案B：进阶版（数据量大时）

```
云服务器：
- 运行爬虫和清洗脚本
- 存储热数据（近期数据）

OSS对象存储：
- 存储冷数据（历史数据）
- 数据备份

数据库（MySQL/PostgreSQL）：
- 结构化存储关键数据
- 支持复杂查询
```

## 数据更新脚本架构

```
scripts/
├── spiders/                    # 爬虫脚本
│   ├── spider_leagues.py       # 联赛数据
│   ├── spider_cups.py          # 杯赛数据
│   └── spider_odds.py          # 赔率数据
│
├── cleaners/                   # 清洗脚本
│   ├── clean_league.py         # 联赛数据清洗
│   ├── clean_cup.py            # 杯赛数据清洗
│   └── validate_data.py        # 数据验证
│
├── sync/                       # 同步脚本
│   ├── upload_to_cloud.py      # 上传到云端
│   ├── merge_to_newdata.py     # 合并到new_data
│   └── backup_to_oss.py        # 备份到OSS
│
└── cron_daily_update.py        # 每日定时更新入口
```

## 建议的实施步骤

1. **第一阶段**：简单部署
   - 云服务器存储数据
   - 本地整理上传
   - rsync同步

2. **第二阶段**：自动化
   - 开发爬虫脚本
   - 配置定时任务
   - 自动更新常规数据

3. **第三阶段**：优化
   - 添加数据库
   - 开发API接口
   - OSS备份
