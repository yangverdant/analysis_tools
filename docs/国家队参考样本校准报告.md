# 国家队参考样本校准报告

生成时间：2026-06-24

## 目标

验证“世界杯比赛是否可以参考对应国家队的友谊赛/国际赛/洲际赛/预选赛历史样本”。

本报告只做离线校准，不改事实数据、不改生产模型。

## 数据与脚本

- 本地脚本：`scripts/audit_national_reference_calibration.py`
- 云端候选事实表：`team_match_facts_candidate_full_20260624`
- 云端生产事实表：`team_match_facts`
- 回测范围：2026-06-13 至 2026-06-23，世界杯，42 场
- 云端候选事实表样本量：754093 行

## 关键结论

国家队历史样本有参考价值，但不是所有玩法都能用。

### 大小球

云端候选事实表结果：

- 有明确国家队大小球方向：36 场
- 命中：22 场
- 命中率：61.1%
- 与当前模型冲突：8 场
- 冲突时国家队方向命中：5 场
- 冲突覆盖净增：+2

阈值扫描：

| 国家队总进球信号偏离盘口 | 冲突场次 | 国家队冲突命中 | 净增 |
| --- | ---: | ---: | ---: |
| 0.10 | 8 | 5/8 | +2 |
| 0.18 | 8 | 5/8 | +2 |
| 0.25 | 7 | 5/7 | +3 |
| 0.35 | 6 | 4/6 | +2 |
| 0.50 | 5 | 3/5 | +1 |

结论：大小球可以进入下一阶段“冲突校验门”测试。建议优先测试 `0.25` 阈值。

### 半全场

- 国家队半场轴：14/42
- 命中率：33.3%

结论：暂时只能作为解释信号，不能进模型权重。

### 胜平负

- 国家队胜平负轴：22/42
- 命中率：52.4%
- 当前模型：24/42，57.1%
- 覆盖后净亏：-2

结论：不能覆盖当前胜平负模型。

### 让球胜平负

最初看起来很高，但复查发现脚本用了 `lottery_matches.handicap_line` 原始值，符号可能和真实让球线不一致。

修正为读取报告里的 signed `goal_line` 后：

- 国家队让球轴：16/42
- 命中率：38.1%
- 当前模型：14/42，33.3%
- 覆盖净增：+2，但绝对命中率仍低

结论：暂不进模型，只保留为诊断信号。这里更像暴露了现有 RQSPF 仍需单独重构，而不是国家队样本已经可靠。

## 可落地规则候选

只针对世界杯/国家队比赛的大小球：

1. 两队国家队样本都不少于 16 场。
2. 国家队总进球信号与真实 O/U 盘口偏离至少 0.25。
3. 国家队方向与当前模型大小球方向冲突。
4. 仅作为候选覆盖，必须走 `run_model_reanalysis_stage.py` 保护门。
5. 若任一保护玩法退步，自动回滚。

## 不进入模型的内容

- 国家队样本不直接覆盖胜平负。
- 国家队样本不直接覆盖半全场。
- 国家队样本不直接覆盖让球胜平负。
- 友谊赛样本不直接替代正式赛样本，只能作为一类可解释证据。

## 下一步

1. 实现大小球国家队冲突校验门，默认关闭，用环境变量打开。
2. 本地回测 2026-06-13 至 2026-06-23 世界杯。
3. 云端用 `team_match_facts_candidate_full_20260624` 回测同范围。
4. 保护门必须保证 `spf/rqspf/bqc/ou/bf` 不退步。
5. 通过后再考虑进入自动化任务。

## 2026-06-24 本地实现回测

已实现默认关闭的国家队大小球冲突闸门：

- 开关：`FOOTBALL_NATIONAL_OU_CONFLICT_GATE_ENABLED=1`
- 事实表：`FOOTBALL_NATIONAL_REFERENCE_FACT_TABLE`
- 阈值：`FOOTBALL_NATIONAL_OU_CONFLICT_BAND=0.25`
- 最小样本：`FOOTBALL_NATIONAL_OU_MIN_SAMPLE=16`

关闭开关回测：

- 范围：2026-06-13 至 2026-06-23，世界杯 42 场
- 候选变化：0
- 结论：默认关闭时不扰动现有分析。

第一次前置闸门回测：

- 大小球：24/42 -> 26/42
- 让球胜平负：19/42 -> 22/42
- 比分：13/42 -> 12/42
- 总体：95/210 -> 99/210
- 结果：因比分保护玩法退步，自动回滚。

这说明国家队样本可以纠偏大小球，但暂时不能让它反向牵动比分、半全场或让球胜平负。

第二次后置 O/U-only 闸门回测：

- 只改变大小球，不改 SPF/RQSPF/BQC/比分候选。
- 大小球：24/42 -> 26/42，57.1% -> 61.9%
- 总体：95/210 -> 97/210，45.2% -> 46.2%
- 变化：8 场大小球，5 场修正，3 场回退。
- 保护玩法退步：0
- 结果：本地保护门通过。

一致性审计：

- `audit_prediction_consistency.py`：42 份报告，issues=0
- `audit_ou_goal_axis.py`：hard_issues=0，soft_issues=9

软问题主要是：国家队样本把 O/U 改成大球，但 Top3 比分仍多落在小球侧。这个并非一定统计错误，因为“大球概率”可以来自多个分散比分组合，而不是 Top3 单点比分；但前端观感容易显得矛盾。下一阶段不能直接隐藏这个问题，需要在“进球区间概率”和“比分单点概率”之间增加解释或更细的比分联动门。

当前结论：

- 后置 O/U-only 闸门可以进入云端候选表回测。
- 前置闸门不能上线。
- 国家队样本暂时只改大小球，不扩散到比分、半全场、让球胜平负。
