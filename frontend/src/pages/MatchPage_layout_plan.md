# MatchPage 布局优化方案

## 当前问题
所有高级分析模块（20+个section-block）在two-columns下方**单列垂直堆叠**，导致页面极长、信息密度低、需要大量滚动。

## 优化方案：分组 + 双列网格

### 布局结构
```
[顶部信息栏] (match-info-bar, 全宽)
[比赛分析头部] (match-analysis: 赛果预测、比分预测、概率条)
[两列核心区域]
  左列: 近期战绩 | 主客场 | 进攻大小球 | Elo实力 | 历史交锋 | 心理压制
  右列: 伤停信息 | 裁判数据 | 近期赛程 | 新闻动态 | 天气影响 | 比赛动态
[高级分析区域 - 新增analysis-grid双列网格]
  ┌─────────────────┬─────────────────┐
  │ AI预测面板       │ ML深度预测       │
  │ (赛果+比分+概率) │ (概率条+模型信息) │
  ├─────────────────┼─────────────────┤
  │ 多模型对比       │ 特征重要性       │
  ├─────────────────┼─────────────────┤
  │ xG分析          │ 泊松分布         │
  ├─────────────────┼─────────────────┤
  │ 让球分析         │ 价值投注         │
  ├─────────────────┼─────────────────┤
  │ 效率对比         │ 爆冷分析         │
  ├─────────────────┼─────────────────┤
  │ 六分之战/战意    │ 赛季走势         │
  ├─────────────────┼─────────────────┤
  │ H2H模式         │ 交锋模式         │
  ├─────────────────┼─────────────────┤
  │ 疲劳对比         │ 轮换风险         │
  ├─────────────────┼─────────────────┤
  │ 裁判影响         │ 场地影响         │
  ├─────────────────┼─────────────────┤
  │ 联赛积分影响     │ 天气详细         │
  ├─────────────────┼─────────────────┤
  │ 换帅效应         │ 阵容分析         │
  ├─────────────────┼─────────────────┤
  │ 比赛前瞻         │ 相关新闻         │
  ├─────────────────┴─────────────────┤
  │ 时区转换 (全宽)                    │
  └───────────────────────────────────┘
```

### 关键CSS
```css
.analysis-grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 16px;
}
.analysis-grid .section-block {
  /* 确保卡片填满 */
}
.analysis-full {
  grid-column: 1 / -1; /* 全宽行 */
}
```

### 响应式
- 移动端: analysis-grid 变为单列

### 分组标题
每个逻辑分组前添加一个分组标题行：
- 🤖 AI预测 → 包含ML深度预测、多模型对比、特征重要性
- ⚽ 进攻分析 → xG、泊松分布、效率对比
- 💰 投注参考 → 让球分析、价值投注、爆冷分析
- 🔥 比赛背景 → 六分之战、战意、赛季走势、H2H模式
- 🏋️ 体能影响 → 疲劳、轮换风险、换帅效应、阵容分析
- 🌍 环境因素 → 裁判、场地、天气、联赛影响、时区
- 📋 前瞻汇总 → 比赛前瞻、相关新闻
